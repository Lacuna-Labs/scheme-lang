# scheme/

**Sakura Scheme language canon.** Everything about the language Sakura speaks, reads, and writes.

## What's here

- **Reference manual** — `SAKURA-SCHEME-1.0-REFERENCE.md` (the language, the verbs, the appendix "Challenges" CS-guts chapters).
- **Book canon** — the locked 16 books, their scope, their ownership. See also `../canon/`.
- **Verb ladders** — 1·3·5 concept-ladder format, removal-test gate, reference webs.
- **Training taxonomy** — data types, atom specs, world-knowledge graph geometry, corpus branches.
- **Compatibility policy** — R5RS/SICP compat strategy (semantic not stylistic), refuse-lists.

## Scope

**In:** anything that defines the *language itself* — its syntax, its verbs, its books, its corpus shape. The canon that both Sakura and human authors follow.

**Out:** the interpreter implementation (that's code, not docs). The trainer runbook (that's `../ops/`). Specific per-project uses of Scheme (those are per-project docs).

## Rule

If Sakura's answer would change based on this doc, it belongs here. If only a human tool depends on it, it probably belongs in `../engineering/` or `../ops/`.
